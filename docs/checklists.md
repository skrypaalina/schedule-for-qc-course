# Чеклісти для тестування (Оновлено)
## Модуль: CRUD викладачів та кафедр (Варіант 5)

### Функціональність (Позитивні сценарії)
- [ ] Створення нової кафедри з унікальною назвою.
- [ ] Додавання викладача з прив'язкою до існуючої кафедри.
- [ ] **Редагування даних користувача** (зміна імені/профілю менеджера).
- [ ] Редагування ПІБ викладача та перевірка збереження змін.
- [ ] Відображення списку викладачів після фільтрації за кафедрою.
- [ ] **Приховування викладача/кафедри** (перевірка статусу "архівний" або "схований").
- [ ] Пошук викладача за прізвищем (частковий збіг).

### Бізнес-логіка та Обмеження
- [ ] **Заборона створення абсолютних тезок** (система не дозволяє два однакових ПІБ).
- [ ] **Блокування видалення викладача**, якщо він вже присутній у розкладі.
- [ ] **Блокування видалення кафедри**, якщо до неї прив'язані викладачі або заняття.
- [ ] Перевірка, що "схований" викладач не відображається у загальному списку для вибору в розклад.

### Негативні сценарії
- [ ] Спроба зберегти викладача з порожнім обов'язковим полем ПІБ.
- [ ] Введення цифр та спецсимволів у поле імені викладача.
- [ ] Введення тексту довжиною понад 50 символів у поле ПІБ (перевірка ліміту).
- [ ] Спроба створити кафедру-дублікат.

### UI/UX
- [ ] Наявність модального вікна підтвердження при видаленні.
- [ ] Відображення зрозумілого повідомлення про помилку при спробі видалити об'єкт із розкладу.
- [ ] Відображення повідомлень про помилки валідації червоним кольором.
- [ ] Адаптивне відображення таблиці викладачів на мобільних пристроях.
